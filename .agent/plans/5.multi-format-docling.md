# Module 5: Multi-Format Support with Docling - Implementation Plan

## Context

Module 5 adds multi-format document support via Docling to the document ingestion pipeline. Currently, only text-based files (.txt, .md) can be properly ingested — binary files like PDF and DOCX produce garbled content because they're read as raw bytes with UTF-8 errors ignored. This module enables parsing of PDF, DOCX, HTML, and Markdown formats using Docling.

**Scope:** Docling only — no unstructured, mistral, or other parsers.

## Phase 2: Document Parser Service

### 2.1 New Parser Service
**File:** `backend/app/services/docling_parser.py` (NEW)

```python
from docling.document_converter import DocumentConverter

class DoclingParser:
    """Parses PDF, DOCX, HTML, Markdown using Docling."""

    SUPPORTED_TYPES = {".pdf", ".docx", ".html", ".htm", ".md"}

    def can_parse(self, filename: str) -> bool:
        """Check if file type is supported."""
        ext = os.path.splitext(filename.lower())[1]
        return ext in self.SUPPORTED_TYPES

    def parse(self, file_path: str, filename: str) -> str:
        """Parse document and return text content."""
        converter = DocumentConverter()
        result = converter.convert(file_path)
        return result.document.export_to_text()
```

Key points:
- Uses `DocumentConverter` from docling
- `export_to_text()` returns plain text
- Returns empty string on parse failure (ingestion continues with warning)
- File extension check determines if Docling should be used

### 3.1 Modify Ingestion Service

Update `process_document()` method:

```
Before (current):
  with open(file_path, "r", encoding="utf-8", errors="ignore") as f:

After (new):

      content = docling_parser.parse(file_path, filename)

      with open(file_path, "r", encoding="utf-8", errors="ignore") as f:
          content = f.read()
```

## Phase 5: File Size Limit (5MB)

### 5.1 Backend File Size Validation
**File:** `backend/app/api/documents.py`

In `upload_document()` endpoint, add file size check:

```python
MAX_FILE_SIZE_MB = 5
MAX_FILE_SIZE_BYTES = MAX_FILE_SIZE_MB * 1024 * 1024

content = await file.read()
file_size = len(content)

if file_size > MAX_FILE_SIZE_BYTES:
    raise HTTPException(
        status_code=413,
        detail=f"File size exceeds maximum allowed size of {MAX_FILE_SIZE_MB}MB"
    )
```

### 5.2 Frontend File Size Validation
**File:** `frontend/src/components/documents/DocumentUpload.tsx`

In `handleUpload()` function, add client-side validation:

```typescript
const MAX_FILE_SIZE_MB = 5;

if (file.size > MAX_FILE_SIZE_BYTES) {
  setError(`File size exceeds maximum allowed size of ${MAX_FILE_SIZE_MB}MB`);
  return;
}
```

Also update the accepted types label to include file size:

## Phase 6: Batch Upload

### 6.1 Frontend Batch Upload Interface
**File:** `frontend/src/components/documents/DocumentUpload.tsx`

Changes:
- Add `multiple` attribute to file input
- Track upload progress for each file
- Show individual status for each file (pending, uploading, completed, failed, duplicate)
- Serial upload (one by one, not parallel) using refs to track queue state

New interface:
```typescript
interface UploadItem {
  id: string
  filename: string
  status: 'pending' | 'uploading' | 'completed' | 'failed' | 'duplicate'
  progress: number
  error?: string
  duplicateOf?: string
}
```

### 6.2 Upload Queue
- Files are uploaded serially (one by one)
- Each file shows its own status and progress bar
- Progress capped at 99% during upload, jumps to 100% on completion
- Uses refs to track queue state (avoid React closure issues)
- Completed uploads trigger onUploadComplete callback after each file
- "Clear completed" button to dismiss all completed items
