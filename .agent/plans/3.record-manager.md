# Module 3: Record Manager - Implementation Plan

**Complexity**: ⚠️ Medium - Requires coordinated changes across database, backend API, ingestion service, and frontend

## Overview

Module 3 adds content-based deduplication to prevent processing duplicate documents. When a user uploads a file, the system will:
1. Compute SHA-256 hash of the raw file content
2. Check if any document with the same hash already exists for this user
3. If duplicate found, return existing document info instead of re-processing
4. If new, proceed with ingestion and store the hash

## Part 2: Backend - Ingestion Service Changes

### 2.1 New Method: `compute_content_hash()`

**File**: `backend/app/services/ingestion_service.py`

Add to `IngestionService`:

```python
import hashlib

def compute_content_hash(self, file_path: str) -> str:
    """Compute SHA-256 hash of file content."""
    sha256_hash = hashlib.sha256()
    with open(file_path, "rb") as f:
        for chunk in iter(lambda: f.read(8192), b""):
            sha256_hash.update(chunk)
    return sha256_hash.hexdigest()
```

### 2.2 New Method: `find_duplicate_document()`

Add to `IngestionService`:

```python
def find_duplicate_document(self, user_id: str, content_hash: str) -> dict | None:
    """Find existing document with same content hash for user."""
    response = self.supabase.table("documents").select(
        "id, filename, file_size, content_hash, created_at"
    ).eq("user_id", user_id).eq("content_hash", content_hash).execute()

    return response.data[0] if response.data else None
```
