# Module 4: Metadata Extraction - Implementation Plan

## Context

Module 4 adds LLM-powered metadata extraction to the document ingestion pipeline and enables filtered retrieval based on extracted metadata. Currently documents are stored with minimal metadata (`{}`), and retrieval is purely vector-similarity based. This module adds:

1. **Structured metadata extraction** using LLM (title, author, date, category, tags, summary, language)
2. **Metadata storage** in existing `documents.metadata` JSONB column
3. **Filtered RAG retrieval** - ability to filter by tag/category before vector search
4. **Frontend display** - show metadata tags on documents, filter UI, metadata detail dialog

**Separation from Module 3:** Module 3 (Record Manager) handles content hashing and deduplication. Module 4 handles metadata extraction and is independent.

## Phase 1: Schema and Metadata Service

### 1.1 Pydantic Schema
**File:** `backend/app/schemas/metadata.py`

```python
class DocumentMetadata(BaseModel):
    title: Optional[str] = None
    author: Optional[str] = None
    date: Optional[str] = None
    category: Optional[str] = None
    tags: list[str] = Field(default_factory=list)
    summary: Optional[str] = None
    language: Optional[str] = None

class MetadataExtractionResult(BaseModel):
    success: bool
    metadata: DocumentMetadata
    error: Optional[str] = None
```

### 1.2 Metadata Extraction Service
**File:** `backend/app/services/metadata_service.py`

- Uses MiniMax LLM to extract metadata from document content
- Truncates content to 4000 chars to avoid excessive tokens
- Handles JSON extraction from LLM responses (including markdown code blocks)
- Tags limited to 3-5 most relevant ones
- Best-effort: extraction failure logs warning but doesn't block ingestion

## Phase 2: Ingestion Integration

### 2.1 Modified Ingestion Pipeline
**File:** `backend/app/services/ingestion_service.py`

**Flow:**
```
Read file → Extract metadata → Chunk → Embed → Store → Update status
```

Metadata extraction happens after reading file content, before chunking. On failure, logs warning and continues.
