## Part 1: Multi-LLM Provider Abstraction

### 1.1 Provider Interface
**New File**: `backend/app/services/llm_providers/base.py`
```python
class BaseLLMProvider(ABC):
    @abstractmethod
    def chat_complete(self, messages: list[dict], stream: bool = False)
        -> LLMResponse | Generator[LLMStreamChunk, None, None]
```

### 1.2 Provider Implementations
| Provider | File | SDK | Notes |
|----------|------|-----|-------|
| MiniMax | `minimax.py` | OpenAI SDK | Primary chat |
| Ollama | `ollama.py` | httpx | Local models |

### 1.3 Factory Pattern
**New File**: `backend/app/services/llm_providers/factory.py`
- Enum: `LLMProviderType` (MINIMAX, OLLAMA)
- Factory creates provider based on type

### 1.4 Config Changes
**Modify**: `backend/app/config.py`
```python
# Default provider selection
default_llm_provider: str = "minimax"

# MiniMax (chat)
minimax_api_key: str
minimax_base_url: str = "https://api.minimax.chat/v1"
minimax_model: str = "MiniMax-M2.7-highspeed"

# Ollama (local)
ollama_base_url: str = "http://localhost:11434"
ollama_model: str = "llama3.2"
```

### 1.5 Chat API Updates
**Modify**: `backend/app/api/chat.py`
- Add `provider` parameter to `stream_chat` request
- Use provider factory to create appropriate LLM service

## Part 2: Database Schema

### 2.1 pgvector Extension
**New Migration**: `backend/scripts/migrations/002_enable_pgvector.sql`
```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 2.2 Documents Table
**New Migration**: `backend/scripts/migrations/003_documents_table.sql`
```sql
CREATE TABLE public.documents (
    id UUID PRIMARY KEY,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    filename TEXT NOT NULL,
    file_path TEXT NOT NULL,
    file_size INTEGER,
    mime_type TEXT,
    status TEXT DEFAULT 'pending' CHECK (status IN ('pending', 'processing', 'completed', 'failed')),
    chunk_count INTEGER DEFAULT 0,
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW(),
    updated_at TIMESTAMPTZ DEFAULT NOW()
);
-- RLS policies + indexes
```

### 2.3 Document Chunks Table
**New Migration**: `backend/scripts/migrations/004_document_chunks_table.sql`
```sql
CREATE TABLE public.document_chunks (
    id UUID PRIMARY KEY,
    document_id UUID REFERENCES public.documents(id) ON DELETE CASCADE,
    user_id UUID REFERENCES auth.users(id) ON DELETE CASCADE,
    chunk_index INTEGER NOT NULL,
    content TEXT NOT NULL,
    embedding VECTOR(1024),  -- MiniMax embedding dimension
    metadata JSONB DEFAULT '{}',
    created_at TIMESTAMPTZ DEFAULT NOW()
);
-- RLS policies + IVFFlat index for vector search
```

### 4.1 Document API Endpoints

## Implementation Order

### Phase 1 (Sequential - Foundation)
1. `backend/app/config.py` - MiniMax + Ollama config

2a. `backend/app/services/llm_providers/` - Provider abstraction (MINIMAX, OLLAMA)
