# Module 7: Additional Tools - Text-to-SQL

## Context

Module 7 requires implementing a Text-to-SQL tool that allows users to query document statistics using natural language (e.g., "What is the average size of my documents?", "Sort by size").

Web Search is not implemented (user confirmed MiniMax's internet search capability is sufficient), but available Web Search components are documented below.

## Complexity: ⚠️ Medium

Requires modifying LLM provider to support tool calling, plus SQL generation and execution logic.

### Phase 3: Tool Router

**New: `backend/app/services/tool_router.py`**
```python
class ToolRouter:
    """
    Routes user queries to appropriate tools.
    """
    def __init__(self, supabase: Client, llm_provider: BaseLLMProvider):
        self.supabase = supabase
        self.llm_provider = llm_provider

    async def route(self, query: str, user_id: str) -> dict:
        """
        Decide which tool to use:
        - Text-to-SQL: Query document statistics
        - RAG: Document content Q&A
        - Fallback: Direct answer
        """
        # Detect if statistical query
        if self._is_statistical_query(query):
            return await self._handle_text_to_sql(query, user_id)

        # Otherwise use RAG
        return None  # Let caller handle
```
